You are a senior Full-Stack Software Engineer, UI/UX Designer, AI Engineer, and Browser Automation Expert.
Your task is to build a complete, production-ready AI-powered Web Novel Audiobook Reader that I can run locally and later host on my own VPS or cloud server.
Do not create a demo or prototype. Build a real, modular, scalable application following software engineering best practices.

Main Objective
The application should allow me to paste the URL of a supported web novel chapter.
Once I click Start Reading, the application should automatically:
1. Open the webpage.
2. Wait until it fully loads.
3. Detect and extract only the novel chapter.
4. Ignore all unrelated page elements.
5. Convert the extracted chapter into speech.
6. Begin narrating from the first sentence to the last sentence without missing any words.
The user should never need to search for a novel within the application. The only required input is the chapter URL.

Continuous Reading Mode (Highest Priority)
The application must support continuous audiobook playback across an entire novel.
This is a core feature and must be implemented as part of the application's workflow.
Workflow
When the user pastes the URL of any chapter (typically Chapter 1) and clicks Start Reading, the application should:
1. Open the provided URL.
2. Extract the chapter title and full chapter content.
3. Narrate the chapter from beginning to end.
4. Detect when the narration has finished.
5. Automatically locate the website's Next Chapter button or link.
6. Click the Next Chapter button programmatically using browser automation.
7. Wait for the next chapter to load completely.
8. Extract the next chapter.
9. Continue narrating without requiring any user interaction.
10. Repeat this process until no additional chapter exists.
The listening experience should feel like a continuous audiobook rather than a collection of separate chapters.

Automatic Chapter Detection
The extraction engine should intelligently identify common navigation elements used by web novel websites, such as:
* Next Chapter
* Next
* 
* ?
* Continue Reading
* Chapter +1
It should also inspect:
* Anchor text
* CSS selectors
* Navigation bars
* Pagination controls
* Common web novel layouts
Design the navigation system to be modular so additional website-specific extractors can be added easily in the future.

End of Novel Detection
The application should automatically stop only when one of the following conditions is met:
* There is no valid Next Chapter button.
* The website explicitly indicates that the novel has ended.
* The next chapter cannot be reached after multiple retry attempts.
* A fatal extraction or navigation error occurs.
When the novel ends, display a message such as:
"You've reached the end of the available chapters."
Then stop playback gracefully.

Continuous Playback Requirements
The transition between chapters should be seamless.
Requirements:
* No manual clicking after the initial Start Reading.
* No page refreshes visible to the user if avoidable.
* Preserve playback settings across chapters.
* Keep the selected voice.
* Keep the selected playback speed.
* Keep the current volume.
* Automatically update the progress indicator.
* Automatically update the chapter title.
* Save reading progress after each completed chapter.
* If the application is restarted, allow resuming from the last completed chapter and playback position.

User Controls During Continuous Reading
The user must be able to interrupt playback at any time.
Supported controls:
* Play
* Pause
* Resume
* Stop
* Skip to Next Chapter
* Return to Previous Chapter
* Forward 10 seconds
* Backward 10 seconds
* Seek anywhere in the current chapter
* Change playback speed while narration is active
* Change voice (applies from the next chapter or after restarting narration)
* Bookmark the current location
* Exit continuous reading mode
These controls must remain responsive even while the browser automation is navigating between chapters.

Intelligent Recovery
If navigation to the next chapter fails:
1. Retry several times using configurable delays.
2. Attempt alternative navigation methods (e.g., different selectors or navigation patterns).
3. If still unsuccessful, stop gracefully and inform the user of the issue without crashing the application.

Performance
Optimize the experience by preloading the next chapter while the current one is being narrated whenever practical.
This minimizes waiting time between chapters and creates a smooth, uninterrupted audiobook experience.
Continuous Reading Mode should be enabled by default, with an optional setting to disable it if the user prefers to read a single chapter only.
